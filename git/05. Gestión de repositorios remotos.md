Los repositorios remotos son versiones de tu proyecto hospedadas en Internet o en una red. 
## Ver tus remotos
Para listar los remotos configurados:
```bash
git remote
```
Para ver las URLs asociadas (lectura y escritura):
```bash
git remote -v
```
## Añadir repositorios remotos
Para asociar un nombre fácil a una URL:
```bash
git remote add [nombre] [url]
```
## Traer y combinar remotos
1. `git fetch [remote-name]`: Descarga los datos que no tienes, pero **no modifica tu trabajo actual**. Es la opción más segura para revisar cambios antes de integrarlos.
2. `git pull`: Descarga y combina automáticamente. Muchos desarrolladores prefieren `git pull --rebase` para mantener un historial limpio y lineal sin commits de *merge* innecesarios.
## Enviar a tus remotos
```bash
git push [nombre-remoto] [nombre-rama]
```
La mayoría de plataformas como GitHub han cambiado el nombre de la rama por defecto de master a **main**.
## Inspeccionar un remoto
Para ver detalles avanzados (ramas rastreadas, qué ocurrirá al hacer pull, etc.):
```bash
git remote show origin
```
## Eliminar y renombrar remotos
- **Renombrar**: `git remote rename pb paul`
- **Eliminar**: Se recomienda usar el comando más descriptivo:
```bash
git remote remove paul
```

## Configurar el repositorio
### 1. Configurar *main* como rama predeterminada globalmente
Para que cada vez que inicies un repositorio nuevo (`git init`) la rama principal se llame `main` automáticamente en lugar de `master`:
```bash
git config --global init.defaultBranch main
```
### 2. Cambiar el nombre en un repositorio existente
Si ya tienes un proyecto con la rama `master` y quienes renombrarla localmente:
```bash
git branch -m master main
```
Si ya habías subido `master` al servidor (como GitHub), deberás subir la nueva rama y borrar la antigua:
```bash
git push -u origin main
git push origin --delete master
```
### 3. Configurar pull con rebase por defecto
Para evitar tu historial se llene de mensajes "*Merge branch 'main' of...*", puedes configurar Git para que siempre intente organizar tus commits linealmente al hacer pull:
```bash
git config --global pull.rebase true
```
