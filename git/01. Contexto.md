## Sobre el control de versiones
Es un sistema que registra los cambios realizados en un archivo o conjunto de archivos a lo largo del tiempo.

Existen tres tipos:
1. Los **sistemas de control de versiones locales** que se almacenan localmente en una base de datos.
2. Los **sistemas de control de versiones centralizados** que se almacenan en un único servidor.
3. Los **sistemas de control de versiones distribuidos** donde los usuarios pueden replicar el repositorio completo, que ayuda a que si un servidor cae, cualquiera de los repositorios disponibles en los usuarios puede ser copiado al servidor con el fin de restaurarlo.
## Git
Entonces, Git es una **sistema de control de versiones distribuido** de código abierto que permite a los desarrolladores rastrear y gestionar cambios en el código fuente de proyectos de software.

Facilita la colaboración al crear "instantáneas" *(snapshots)* del proyecto para guardar su historial y permitir la creación de **ramas** para trabajar de forma independiente, todo con velocidad y eficiencia, incluso sin conexión a internet.
### ¿Cómo funciona?
1. **Área de trabajo**: Archivos modificados en tu proyecto.
2. **Área de preparación** *(Staging Area)*: Archivos listos para ser incluidos en la próxima confirmación.
3. **Confirmación** *(Commit)*: Se toma una "foto" *(snapshot)* de los archivos preparados y se guarda en el repositorio local con su historial.
4. **Repositorio (Local/Remoto)**: El repositorio local contiene todo el historial; puede sincronizar con un servidor remoto (como GitHub, GitLab) para compartir y colaborar.

En resumen, **Git es esencial para el desarrollo colaborativo, ya que ofrece una forma robusta y eficiente de gestionar la evolución de un proyecto de software**.
