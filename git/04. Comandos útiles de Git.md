## Eliminar archivos
Puedes eliminar archivos rastreados (commiteados) con el comando `git rm <nombre_de_archivo>`, pero si quieres mantener el archivo en el disco duro pero sin que Git  lo siga rastreando, puedes utilizar `git rm --cached <nombre_de_archivo>`.
## Cambiar el nombre de los archivos
Si quieres renombrar un archivo Git, puedes ejecutar `git mv file_from file_to`
## Ver historial de confirmaciones
El comando `git log` lista las confirmaciones hechas sobre ese repositorio en orden cronológico inverso (más recientes se muestren al principio).

También puedes usar la opción `-2` que hace que se muestren únicamente las dos últimas entradas del historial.

Puedes imprimir cada confirmación en una única linea:
```bash
git log --pretty=oneline
ca82a6dff817ec66f44342007202690a93763949 changed the version number
085bb3bcb608e1e8451d4b2432f8ecbe6306e7e7 removed unnecessary test
a11bef06a3f659402fe7563abf99ad00de2209e6 first commit
```
Las opciones temporales como `--since` (desde) y `--until` (hasta) resultan muy útiles (son perfectas para reportes), por ejemplo, este comando lista todas las confirmaciones hechas durante las dos últimas semanas:
```bash
git log --since=2.weeks
``` 
## Deshacer cosas
### Corregir el último commit (`--amend`)
Si confirmas y notas que olvidaste un archivo o te equivocaste en el mensaje, puedes usar:
```bash
git commit --amend -m "nuevo mensaje"
```

Si olvidaste un archivo: `git add archivo_olvidado` y luego `git commit --amend`.

Solo recuerda: **nunca hagas amend en commits que ya subiste al servidor (push)**.
### Deshacer un archivo preparado *(unstaging)*
Si usaste `git add .` por error y quieres sacar un archivo del área de preparación sin borrar tus cambios, el comando moderno es:
```bash
git restore --staged CONTRIBUTING.md
```
### Deshacer un archivo modificado *(discard changes)*
Si hiciste cambios en un archivo pero quieres descartarlos y volver a cómo estaba en el último commit:
```bash
git restore CONTRIBUTING.md
```
**¡Cuidado!** Esta acción es irreversible para cambios que no han sido confirmados.
### El "salvavidas": git reflog
Si borras un commit por error, usas mal un `--amend` o pierdes el rastro de una rama, Git guarda un registro de todos los movimientos del HEAD:
```bash
git reflog
```
Este comando te permite encontrar el SHA (ID) de casi cualquier estado anterior para recuperarlo.
