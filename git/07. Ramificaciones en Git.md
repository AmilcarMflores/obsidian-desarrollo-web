- Una rama es un **apuntador móvil** que apunta a una confirmación (commit) específica.
- El nombre estándar de la rama por defecto ha pasado de `master` a `main`.
- Cada vez que confirmas cambios, el apuntador de la rama actual avanza automáticamente hacia el nuevo commit.
- El apuntador especial **HEAD** es el que le indica a Git en qué rama estás trabajando actualmente.
## Crear una rama nueva
```bash
git branch testing
```
Para ver tus ramas y hacia dónde apunta el HEAD, usa:
```bash
git log --oneline --decorate
```
## Cambiar de rama
```bash
git switch testing  # Forma moderna y recomendada
```
Para crear una rama y saltar a ella en un solo paso:
```bash
git switch -c nueva-rama  # Reemplaza a 'git checkout -b'
```
## Fusionar ramas (*merge*)
Para traer los cambios de una rama (ej. `feature`) a tu rama principal:
1. Te sitúas en la receptora: `git switch main`
2. Fusionas: `git merge feature`

Si Git puede mover el apuntador directamente al final de la otra rama sin bifurcaciones, verás el mensaje *"Fast-forward"*.
## Resolver conflictos de *merge*
Cuando Git encuentra cambios contradictorios en la misma linea de un archivo, se detiene y te pide ayuda.
1. Abre el archivo y buscas las marcas `<<<<<<<`, `=======` y `>>>>>>>`.
2. Edita el archivo dejando el código final deseado.
3. **Marcar como resuelto:** `git add <archivo>
4. **Finalizar:** `git merge --continue` (o `git commit`).
## Eliminar una rama
- **Borrado seguro:** `git branch -d nombre-rama` (solo funciona si la rama ya fue fusionada).
- **Borrado forzoso:** `git branch -D nombre-rama` (úsalo si quieres descartar la rama y sus experimentos por completo).
