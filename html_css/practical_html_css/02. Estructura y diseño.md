## Elementos estructurales en HTML5
HTML5 introdujo etiquetas semánticas que permiten **organizar y dar sentido al contenido** de una página web.

![[Pasted image 20250830211633.png|centrear-imagen|reducir-imagen]]

### header
- Representa el **encabezado** de una página o sección.
- Puede contener: logotipo, título, navegación principal, barra de búsqueda.
### footer
- Representa el **pie de página**.
- Suele incluir: derechos de autor, enlaces legales, redes sociales, información de contacto.
### section
- Define una **sección temática** de un documento.
- Útil para dividir contenido en partes lógicas (ej. capítulos, bloques de contenido).
### article
- Representa un **contenido independiente y reutilizable** (puede existir fuera de la página actual).
- Ejemplos: publicaciones de blog, artículos de prensa, comentarios de usuarios.

```html
<article class="post">   
	<h2>Titular del artículo</h2>   
	<p>Contenido del artículo...</p> 
</article>
```

### nav
- Agrupa enlaces de **navegación**.
- Generalmente se coloca en el `header` o en un `aside`.
### aside
- Contenido **complementario o relacionado** con el principal.
- Usado en barras laterales, publicidad o enlaces relacionados.
### div
- Elemento **genérico** para agrupar contenido sin significado semántico.
- Se usa junto con clases o identificadores para aplicar estilos o scripts.

## Diseños de páginas con CSS
Existen varias formas de diseñar páginas web. Las tres técnicas más usadas son:
- **Float** (diseños flotantes, método clásico).
- **Flexbox** (diseños flexibles, más moderno).
- **Grid** (diseños en cuadrícula, el más potente y reciente).
### Float-based layouts (diseños flotantes)
Un método **antiguo**, pero todavía útil para comprender cómo se diseñaban páginas antes de Flexbox y Grid.
#### Propiedad `float`
Permite colocar un elemento a la izquierda o derecha dentro de su contenedor:

```css
float: right;
float: left;
```

#### Propiedad `width`
Define el ancho de los elementos (en píxeles o porcentajes):

```css
width: 100px; 
width: 25%;
```

**Inconveniente con float:**
Los elementos con `float` pueden “salirse” del contenedor, afectando al diseño. Una solución rápida podría ser:

```css
section {   
	overflow: hidden; 
}
```

### Flex-based layouts (diseños flexibles)
Flexbox es mucho más moderno y facilita **alinear, distribuir y adaptar elementos** a diferentes pantallas con poco código.
#### Flex container
Activamos un contenedor flexible así:

```css
display: flex;
```

- Por defecto, los elementos hijos se alinean en **fila horizontal**.
- Para permitir que salten a **múltiples filas**:

```css
flex-wrap: wrap;
```

#### Flex items
Para definir el tamaño base de cada elemento hijo:

```css
flex-basis: 25%; /* 4 elementos por fila */
```

### Grid-based layouts (diseños en cuadrícula)
Grid es el método más **potente y flexible**, ideal para crear estructuras complejas de filas y columnas.
#### Grid container
Para activar un contenedor grid:

```css
display: grid;
```

Definimos el número y tamaño de columnas:

```css
grid-template-columns: auto auto auto auto; /* 4 columnas */
```

También podemos usar fracciones (`fr`) en lugar de valores fijos:

```css
grid-template-columns: 1fr 1fr; /* 2 columnas iguales */
```

#### Grid items
A diferencia de `float` y `flex`, en Grid no siempre es necesario especificar anchos, ya que los **elementos se ajustan automáticamente a la cuadrícula**.
## The Box Model
Cada elemento HTML se puede imaginar como un **cuadro** formado por varias capas: **content box**, **padding**, **border** y **margin**. Estas capas determinan cómo se muestra y se separa un elemento dentro de la página:

- **Content box:** área donde se muestra el contenido real, como texto o imágenes. 
- **Padding:** espacio interno entre el contenido y el borde, que genera separación visual.
- **Border:** rodea el padding y el contenido; puede tener distintos estilos, grosores y colores. Por defecto no se ve, se debe definir explícitamente con: **ancho, estilo y color**.
- **Margin:** espacio externo que separa el elemento de otros elementos en la página.
