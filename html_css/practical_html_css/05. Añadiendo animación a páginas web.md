## Transiciones CSS (Transitions)
Las transiciones permiten cambios suaves entre estados de un elemento.
```css
.elemento {
  propiedad-inicial: valor;
  transition: propiedad duración timing-function delay;
}

.elemento:hover {
  propiedad-final: valor;
}
```
La propiedad transition se describe dentro del valor original, no en el cambiado.
### Propiedades de transición
| Propiedad                    | Descripción               | Valores                                   |
| ---------------------------- | ------------------------- | ----------------------------------------- |
| `transition-property`        | Qué propiedad animar      | `all`, `none`, nombre de propiedad        |
| `transition-duration`        | Duración de la animación  | `0.5s`, `200ms`                           |
| `transition-timing-function` | Curva de velocidad        | `ease`, `linear`, `ease-in-out`           |
| `transition-delay`           | Retardo antes de comenzar | `0.5s`, `200ms`                           |
| `transition`                 | shorthand                 | `property duration timing-function delay` |
### La regla @starting-style
- Resuelve un problema común: las transiciones no se activan en el primer renderizado de un elemento o cuando cambia de display: none a visible.
- **@starting-style** es una nueva regla CSS que permite definir estilos iniciales para elementos que aparecen por primera vez o cambian su estado de visualización. 
- Es especialmente útil para crear animaciones de entrada y salida suaves.
```css
@starting-style {
  .elemento {
    opacity: 0;
  }
}

.elemento {
  opacity: 1;
  transition: opacity 0.5s;
}
```
La transición se aplica desde el momento que el elemento se hace visible.
## Posicionamiento CSS
### Tipos de Position
1. **position: static** Este es el valor predeterminado. No nos permitirá usar las propiedades top, bottom, left y right.
2. **position: relative** Esta propiedad nos permite usar las propiedades top, bottom, left y right para moverlos desde su posición original.
3. **position: fixed** Puedes usar las propiedades tblr, y lo que le hace diferente es que el elemento permanecerá en el mismo lugar incluso cuando se desplaza la página.
4. **postion: absolute** Con este valor, los elementos pueden superponerse a otros elementos, como si estuvieran en una capa por encima del documento original.
5. **position: sticky** Es una combinación de relative y fixed.
---
### z-index
- Cuando asignamos la propiedad `position` a un elemento, nos permite utilizar la propiedad z-index.
- z-index gobierna el orden de apilamiento de un elemento, decidiendo si debe estar posicionado delante o detrás de otros elementos.
- Su valor puede ser positivo entero (capas superiores) o negativo (capas inferiores), siendo 0 a menudo el punto de referencia dentro del código de la aplicación.

## Propiedades de Efectos Visuales
### Opacity
- Controla la transparencia de un elemento.
- Opera en una escala que va de 0.0 (completamente invisible) a 1 (totalmente visible).
### Filter: Blur
Aplica efecto de desenfoque:
```css
.image-container:nth-child(2) img {
	 filter: blur(3px); 
}
```
## Reglas @keyframes y propiedades de animación en CSS
### ¿Qué son?
Las animaciones en CSS permiten que un elemento cambie sus estilos de manera **gradual y controlada**. Se definen en dos partes:
1. **Propiedades de animación (`animation` y subpropiedades):** controlan el **tiempo**, la **velocidad**, cuántas veces se repite, etc.
2. **Regla @keyframes:** define los **estados visuales** de la animación (qué cambia y cómo).
### Propiedades principales de animación
| **Propiedad**                  | **Valores aceptados**                                                                | **Descripción**                                             |
| ------------------------------ | ------------------------------------------------------------------------------------ | ----------------------------------------------------------- |
| `animation` *(shorthand)*      | `name duration timing-function delay iteration-count direction fill-mode play-state` | Permite escribir todas las propiedades en una sola línea.   |
| `animation-name`               | Nombre de un `@keyframes` (ej: `slide-in`)                                           | Conecta con la animación definida en `@keyframes`.          |
| `animation-duration`           | `s` o `ms` (ej: `2s`, `500ms`)                                                       | Duración de 1 ciclo de la animación.                        |
| `animation-timing-function`    | `ease`, `linear`, `ease-in`, `ease-out`, `steps()`, `cubic-bezier()`                 | Controla la curva de velocidad.                             |
| `animation-delay`              | `s` o `ms`                                                                           | Tiempo antes de que empiece la animación.                   |
| `animation-iteration-count`    | `número` o `infinite`                                                                | Veces que se repite la animación.                           |
| `animation-direction`          | `normal`, `reverse`, `alternate`, `alternate-reverse`                                | Dirección de reproducción.                                  |
| `animation-fill-mode`          | `none`, `forwards`, `backwards`, `both`                                              | Estilos que quedan antes o después de la animación.         |
| `animation-play-state`         | `running`, `paused`                                                                  | Ejecutar o pausar animación.                                |
| `animation-timeline` *(nuevo)* | `auto`, `scroll()`, `view()`, etc.                                                   | Define la línea de tiempo (ej: animación ligada al scroll). |
#### Orden en el shorthand
Cuando usamos la abreviatura `animation`, el orden recomendado es:
1. `duration` (duración)  
2. `timing-function` (curva de velocidad)  
3. `delay` (retraso)  
4. `iteration-count` (repeticiones)  
5. `direction` (dirección)  
6. `fill-mode` (modo de relleno)  
7. `play-state` (estado)  
8. `name` (nombre de la animación)  

Ejemplo shorthand:
```css
animation: 3s ease-in 1s 2 reverse both paused slidein;
```
### La regla @keyframes
`@keyframes` define **qué cambia** en la animación.  Se puede usar con palabras clave (`from` / `to`) o porcentajes (`0%`, `50%`, `100%`).

Ejemplo:
```css
@keyframes blurAnimation {
    from { filter: blur(0px); }
    to   { filter: blur(10px); }
}
div {
    width: 200px;
    height: 200px;
    background: lightblue;
    animation: blurAnimation 2s ease-in-out infinite alternate;
}
```
Este div se desenfoca de 0px a 10px y luego regresa, en bucle infinito. 

Ejemplo práctico: Spinner de carga:
```css
.container {
    display: flex;
    justify-content: center;
    align-items: center;
    height: 100vh;
}

img {
    width: 50px;
    height: 50px;
    animation: loadSpinner 1s linear infinite;
}

@keyframes loadSpinner {
    0%   { transform: rotate(0deg); }
    100% { transform: rotate(360deg); }
}

```
La imagen rota constantemente simulando un spinner de carga.
