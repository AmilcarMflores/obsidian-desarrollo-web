## Accesibilidad Web (a11y)
Son prácticas de desarrollo para eliminar barreras y permitir que personas con distintas capacidades puedan usar la web de manera efectiva.
### ¿A quién beneficia?
Beneficia a un amplio espectro de usuarios:
- Personas con discapacidades visuales que usan lectores de pantalla.
- Personas con discapacidades auditivas que necesitan subtítulos o alternativas para el audio.
- Personas con discapacidades físicas que navegan por teclado o necesitan controles grandes.

Implementar a11y mejora la usabilidad general para todos los usuarios y beneficia el SEO.
### Estándares y herramientas Clave
- WAI (Web Accessibility Initiative): La iniciativa del W3C que desarrolla los estándares, como las WCAG.
- ARIA (Accesible Rich Internet Applications): Un conjunto de atributos (roles, propiedades, estados) para mejorar la accesibilidad de componentes dinámicos. No sustituye al HTML semántico; se debe usar solo cuando sea necesario.
## Accesibilidad en Imágenes
### La importancia del texto alternativo (alt)
El atributo `alt` es fundamental porque describe el contenido y la función de una imagen a los usuarios que no pueden verla, como aquellos que utilizan lectores de pantalla.

Ejemplo (imagen informativa):
![[Pasted image 20250907155836.png|centrear-imagen]]

```html
<img src="bar-chart.png" alt="Bar graph of profits for 2019 (£40,000), up £20,000 from 2018 (£20,000)">
```

![[Pasted image 20250907160015.png|centrear-imagen]]
### Guías rápidas para alt:
1. **Imagen Informativa:** Para imágenes que transmiten información importante (como gráficos, diagramas o fotos relevantes), el texto `alt` debe describir el contenido y la información clave de manera concisa.
2. **Imagen Decorativa:** Si la imagen es puramente decorativa y no aparta información relevante, se debe usar `alt=""`. Esto indica al lector de pantalla que ignore el elemento, evitando ruido y mejorando la experiencia.
3. **Imagen como Enlace o Botón:** Cuando una imagen funciona como un enlace o botón, el `alt` debe describir la acción que se realizará al interactuar con ella, no su apariencia.

Ejemplos:
- `alt="Buscar`
- `alt="Enviar formulario`
- `alt="Ir a la página de inicio"`

4. **Descripciones Largas o Complejas:** Si la descripción necesaria es muy larga o detallada (como en gráficas complejas), se debe:
	- Proporcionar un resumen conciso en el atributo `alt`.
	- Incluir una descripción completa en el texto circundante o en una página aparte.
	- Vincular la imagen a esa descripción usando `aria-describedby` o un enlace (`<a>`) para ofrecer más detalles.
## Haciendo los Formularios Accesibles
Para garantizar que los formularios sean accesibles, es fundamental asegurar que cada campo de entrada esté correctamente asociado con su etiqueta de texto. Las técnicas clave para lograrlo son el uso de **`<label>`** y la agrupación con **`<fieldset>`**.
### Asociando Etiquetas con Campos
La asociación entre un texto descriptivo y un campo de formulario se puede realizar de dos maneras:
1. **Usando el atributo `for` en `<label>`:** Se utiliza el elemento `<label>` independientemente con el atributo `for`. El valor de este atributo debe coincidir exactamente con el valor del atributo `id` del campo de entrada:
```html
<label for="first-name">Nombre:</label>
<input type="text" id="first-name" name="first-name">
```

2. **Envolviendo el campo con `<label>`:** Se coloca el campo de entrada dentro del elemento `<label>`. Esto crea una asociación implícita, sin necesidad de usar `for` o `id`:
```html
<label>
  Nombre:
  <input type="text" name="first-name">
</label>
```
### Agrupando Campos con `<fieldset>` y `<legend>`
Cuando un formulario crece y contiene múltiples secciones o grupos de opciones, es necesario organizarlo de manera significativa. Para esto se utilizan los elementos **`<fieldset>`** y **`<legend>`**:
- **`<fieldset>`:** Se utiliza para agrupar semánticamente un conjunto de campos relacionados dentro de un formulario. Actúa como un contenedor que une visual y lógicamente los elementos.
- **`<legend>`:** Proporciona un título o descripción para el grupo definido por el `<fieldset>`.

## Entendiendo la Accesibilidad del Teclado
La navegación mediante teclado es un pilar fundamental de la accesibilidad web, utilizada por personas con discapacidades motoras, usuarios avanzados que prefieren no usar el mouse y cualquier persona en situaciones donde el ratón no es práctico.
### Navegación Básica con Teclado
- **`Tab`:** Avanza el foco al siguiente elemento interactivo en la página (enlaces, botones, campos de formulario).
- **`Shift + Tab`:** Retrocede el foco al elemento interactivo anterior.
- **`Enter` o `Space`:** Activa el elemento que currently tiene el foco (hace clic en un enlace o botón con `Enter`).
- **`Space`:** Específicamente se usa para alternar (check/uncheck) casillas de verificación (checkboxes) y seleccionar botones de radio (radio buttons).
### Indicador de Foco Visible
Para que esta navegación sea efectiva, los usuarios deben poder ver claramente qué elemento tiene actualmente el foco. Esto se logra aplicando estilos específicos usando la pseudo-clase **`:focus`**.
## Haciendo el Movimiento Accesible
Las animaciones y transiciones pueden mejorar la experiencia para muchos usuarios, pero para otros—especialmente aquellos con vértigo, epilepsia o trastornos vestibulares—pueden ser una fuente de distracción, mareos, náuseas o incluso desencadenar convulsiones.
### La Media Query `prefers-reduced-motion`
Para respetar las preferencias y necesidades de estos usuarios, CSS ofrece la media query **`prefers-reduced-motion`**. Esta detecta si el usuario ha configurado en su sistema operativo la preferencia de reducir el movimiento animado.

Los dos valores posibles son:
- **`no-preference`:** Indica que el usuario no ha activado ninguna preferencia para reducir animaciones. Es el valor por defecto.
- **`reduce`:** Indica que el usuario ha solicitado explicitamente minimizar la cantidad de movimiento o animación.
#### Cómo Implementarla
La práctica recomendada es escribir primero los estilos sin animación o con una animación sutil, y luego sobrescribirlos para los usuarios que no tienen preferencia por reducir el movimiento.
## Herramientas de Accesibilidad
Para garantizar que nuestras páginas web sean accesibles, es fundamental utilizar herramientas que nos ayuden a identificar y corregir problemas. Existen muchas herramientas, tanto integradas en los navegadores como desarrolladas por terceros, que automatizan y facilitan la evaluación de la accesibilidad.
### Axe DevTools
Desarrollada por **Deque Systems**. Esta herramienta está diseñada para detectar una amplia gama de problemas de accesibilidad de manera eficiente.

![[Pasted image 20250907233630.png|centrear-imagen|reducir-imagen]]
#### ¿Cómo se utiliza?
1. **Instalación:** Axe DevTools está disponible como una extensión del navegador.
2. **Análisis:** Una vez instalado, abre las Herramientas de Desarrollo (DevTools) de tu navegador. Verás una nueva pestaña llamada "axe DevTools". Al hacer clic en el botón **"Full Page Scan"**, la herramienta analizará todo el marcado HTML de tu página.
3. **Resultados:** Axe generará un informe detallado que categoriza los problemas encontrados por su nivel de gravedad (crítico, serio, moderado, menor).

**La gran ventaja de Axe** es que no solo se limita a señalar problemas; también **proporciona recomendaciones específicas y soluciones** para cada uno, explicando por qué es importante y cómo corregirlo.
Es importante recordar que, si bien las herramientas automáticas como Axe son extremadamente útiles, **no pueden captar todos los problemas de accesibilidad**. Una evaluación completa siempre debe complementarse con pruebas manuales, como la navegación real usando solo el teclado y pruebas con lectores de pantalla.
