## Diseño Mobile First
* ==Estrategia de diseño que prioriza la experiencia en dispositivos móviles antes que la de de escritorio.==
- Ya que desde **2016**, el tráfico móvil superó al de escritorio.
- Empezar el CSS con estilos base para móviles y luego añadir media queries para pantallas mayores.
## Meta viewport
Etiqueta HTML que controla cómo se escala y muestra la página en dispositivos con diferentes tamaños de pantalla.

Ejemplo:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
- `width=device-width` → adapta el ancho al del dispositivo.
- `initial-scale=1.0` → establece el zoom inicial al 100%.
## Media queries (CSS)
* ==Son reglas CSS que aplican estilos condicionales según características del dispositivo==; piedra angular del Responsive Web Design. 
* Muy usado es ancho/alto de pantalla, orientación, resolución/densidad de píxeles, tipo de medio (screen, print).
* Su objetivo es que **un solo sitio se vea bien en cualquier dispositivo**.
### Estructura básica
```css
@media media-type and (condición) {
  /* estilos solo si la condición se cumple */
}
```
- `media-type`: `screen`, `print`, `all`.
- `condición`: por ejemplo `(min-width: 600px)` u`(orientation: landscape)`.

Ejemplo práctico:
```css
@media screen and (min-width: 600px) and (max-width: 900px) {
   .container {
        width: 90%;   
   } 
}
```
## Valores responsivos en CSS
Son valores que cambian según el tamaño del viewport, el contenedor o las preferencias del usuario para lograr interfaces adaptables.
### Unidades de tipografía responsiva
- **rem**: relativo al tamaño de fuente del elemento raíz (por ejemplo, si :root{ font-size: 16px } entonces 1rem = 16px). Útil para escalado global consistente.
- **em**: relativo al tamaño de fuente del elemento padre (si un contenedor tiene font-size: 20px, 1em = 20px dentro de ese contenedor). Útil para escalado local pero puede acumularse (encadenamiento).
### Unidades de tamaño responsivo (vh / vw)
- **vh**: 1vh = 1% de la altura del viewport. 100vh = altura completa del viewport.
- **vw**: 1vw = 1% del ancho del viewport. 100vw = ancho completo del viewport.

Ejemplos:
```css
.hero { height: 100vh; } /* sección full-screen */
.box { width: 50vw; }    /* mitad del ancho de la ventana */
```

Precauciones:
- Evita usar 100vh en móviles sin ajustes (la UI de navegador puede cambiar la altura real). Considera calcular con CSS env(safe-area) o usar JS para ajustar en casos críticos.
## Imágenes responsivas
### Width / max-width con porcentajes
- **width:** 100% - la imagen escala con el contenedor.
- **max-width:** 100% - evita que la imagen supere el ancho del contenedor manteniendo la relación de aspecto.
```css
img { width: 100%; max-width: 100%; height: auto; display: block; }
```
### Elemento HTML picture
Permite servir diferentes archivos según condiciones (media queries) o formatos (webp, avif):
```html
<picture>
  <source srcset="image-large.webp" type="image/webp" media="(min-width: 800px)">
  <source srcset="image-medium.jpg" media="(min-width: 500px)">
  <img src="image-small.jpg" alt="Responsive Image">
</picture>
```
### srcset y sizes
- srcset: lista de archivos con su ancho en px o descriptor de densidad.
- sizes: indica al navegador qué ancho de imagen (css) se usará en cada condición para que seleccione la mejor entrada de srcset.

```html
<img
  src="example-small.jpg"
  srcset="example-small.jpg 300w, example-medium.jpg 600w, example-large.jpg 1200w"
  sizes="(max-width: 600px) 300px, (max-width: 900px) 600px, 1200px"
  alt="Example Image">

```
Regla rápida: sizes describe el ancho renderizado (en CSS px) que ocupará la imagen en cada rango; el navegador escoge la fuente en srcset más apropiada.
### Imágenes de fondo en CSS
- background-size: cover -- escala la imagen para cubrir el contenedor; puede recortar.
- background-size: contain -- escala la imagen para que quepa completa dentro del contenedor; puede dejar espacio vacío.
```css
.responsive-bg {
  background-image: url('example-small.jpg');
  background-size: cover;
  background-position: center;
  height: 50vh;
}
@media (min-width: 600px) {
  .responsive-bg { background-image: url('example-medium.jpg'); }
}
@media (min-width: 1200px) {
  .responsive-bg { background-image: url('example-large.jpg'); }
}

```
### Lazy loading
- Atributo loading="lazy" difiere la carga de imágenes fuera de pantalla.
```html
<img src="example.jpg" alt="Example Image" loading="lazy">
```

Consideraciones:
- Mejora rendimiento; no usar en imágenes críticas del above-the-fold.
- Para mayor control y compatibilidad, considera IntersectionObserver.
## CSS Flexbox
Sistema de diseño unidimensional que distribuye espacio y alinea elementos dentro de un contenedor según el espacio disponible:
- Flex container / flex items: el elemento padre se convierte en contenedor flexible con `display: flex` o `display: inline-flex`; los hijos son flex items.
- Main axis / Cross axis: el eje principal (main axis) es la dirección en la que se colocan los items; el eje cruzado (cross axis) es perpendicular al principal. Por defecto `flex-direction: row` -> main axis horizontal.
### Propiedades del contenedor (flex container)
- `display`: flex, inline-flex
- `flex-direction`: row | row-reverse | column | column-reverse
- `flex-wrap`: nowrap | wrap | wrap-reverse
- `justify-content`: flex-start | flex-end | center | space-between | space-around | space-evenly  (alinea a lo largo del main axis.)
- `align-items`: flex-start | flex-end | center | baseline | stretch  (alinea a lo largo del cross axis).
- `align-content`: flex-start | flex-end | center | space-between | space-around | stretch (alinea líneas de items cuando hay múltiples filas/columnas [solo aplica si hay espacio extra en el cross axis]).
### Propiedades de los items (flex items)
- `order` — entero que redefine el orden visual (valor por defecto 0).
- `flex-grow` — número que indica cuánto puede crecer un item relativo a los demás (0 por defecto).
- `flex-shrink` — número que indica cuánto puede reducirse un item cuando falta espacio (1 por defecto).
- `flex-basis` — tamaño inicial del item antes de distribuir espacio (por defecto `auto`).
- `flex` — shorthand: `flex-grow flex-shrink flex-basis` (ej. `flex: 1 0 200px`).
- `align-self` — permite anular `align-items` para un item: `auto`, `flex-start`, `flex-end`, `center`, `baseline`, `stretch`.
## Diseño imprimible (print CSS)
Estilos que se aplican únicamente cuando el usuario imprime o previsualiza la impresión, para adaptar la presentación al medio papel (o PDF).
### Cómo aplicar estilos para impresión
- Hoja de estilo separada (HTML):
```html
<link rel="stylesheet" media="print" href="print.css">
```

- Media query dentro de la misma hoja:
```css
@media print { /* estilos específicos para impresión */ }
```
### Buenas prácticas para print CSS
- Oculta elementos no necesarios: navegaciones, banners, elementos interactivos.
```css
@media print { 
	nav, .no-print, .ads { display: none !important; } 
}
```

- Asegura legibilidad: colores de texto contrastados, tamaño de fuente legible, márgenes adecuados.
```css
@media print {
  body { color: #000; background: #fff; font-size: 12pt; margin: 1in; }
}
```

- Controla saltos de página:
```css
@media print {
  h1, h2 { page-break-after: avoid; }
  .page-break { page-break-before: always; }
}
```

- Usa unidades impresas (pt, in, cm) cuando necesites medidas físicas exactas; usa px/rem para referencias relativas cuando no requieras precisión física.
- Asegura que imágenes y fondos críticos se impriman (algunos navegadores ignoran background-images por defecto); considera usar imágenes inline o reglas específicas para `-webkit-print-color-adjust: exact;`.
