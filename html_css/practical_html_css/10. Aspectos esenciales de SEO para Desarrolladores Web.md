SEO es la práctica de mejorar un sitio web para aumentar su visibilidad en las Páginas de Resultados del Motor de Búsqueda *(SERPs)*.

El SEO implica una gama de técnicas y mejores prácticas, incluyendo la investigación de palabras clave, optimización *on-page*, SEO técnico y construcción de enlaces *(link building)*.
### Conceptos clave de SEO
- **Investigación de palabras clave *(Keyword research)***: Identificar los términos y frases que los visitantes potenciales usan para encontrar contenido relevante es crucial.
- **Optimización *On-page***: Optimizar páginas individuales para posicionarse más alto y ganar más tráfico relevante es muy importante. Esto incluye etiquetas de título, meta descripciones, etiquetas de encabezado y la colocación de palabras clave.
- **SEO técnico**: Mejorar los aspectos técnicos de un sitio web, como la velocidad del sitio, la compatibilidad con móviles *(mobile-friendliness)*, la indexación, la capacidad de rastreo *(crawlability)* y la seguridad.
- **Construcción de enlaces *(link building)***: Es importante adquirir hipervínculos de otros sitios web hacia el tuyo. Los *backlinks* de alta calidad pueden mejorar significativamente la autoridad y el ranking de un sitio.
- **Creación de contenido**: Producir contenido valioso y de alta calidad que responda a las consultas de los usuarios y fomente la interacción y el compartir es una parte importante de la optimización SEO.
## El impacto de HTML  y CSS en el SEO
- **Etiquetas HTML semánticas**: Las etiquetas HTML semánticas (tales como `<header>`, `<footer>`, `<article>` y `<section>`) ayudan a los motores de búsqueda a entender la estructura y el contenido de tus páginas web. **El uso adecuado de estas etiquetas facilita que los motores de búsqueda indexen y clasifiquen tu contenido apropiadamente**.
- **Etiquetas de título**: Una etiqueta `<title>` bien elaborada con palabras clave relevantes puede mejorar significativamente el ranking de tu página y la Tasa de Clics *(CTR)* en los resultados de búsqueda.
- **Meta descripciones**: Las meta descripciones (usando la etiqueta `<meta>`) proporcionan un resumen del contenido de la página. Aunque este no es un factor de ranking directo, una meta descripción convincente puede mejorar el *CTR*, influenciando indirectamente los rankings. Un ejemplo:
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <title>Top SEO Tips for Web Developers</title>
    <meta
        name="description"
        content="Discover the best SEO practices for
                web developers to enhance site
                visibility and performance in search
                engines."
    >
</head>
</html>
```
- **Etiquetas de encabezado**: Las etiquetas de encabezado (`<h1>`, `<h2>`, `<h3>`, etc.) estructuran tu contenido e indican su jerarquía. El uso adecuado de etiquetas de encabezado mejora la legibilidad y ayuda a los motores de búsqueda a entender los temas principales de tu contenido. 
- **Enlazado interno**: Los enlaces entre páginas en tu sitio ayudan a establecer la arquitectura y jerarquía del sitio. Un enlazado interno efectivo distribuye la equidad del enlace (poder de ranking) a través de tu sitio, ayudando en la indexación y el ranking de las páginas.
- **Estructura de URL**: URLs limpias y descriptivas facilitan que los usuarios y los motores de búsqueda entiendan el contenido de la página. Incluir palabras clave en las URLs puede mejorar los *CTRs* y los rankings. Un ejemplo:
```html
<body>
    <header>
        <h1>Top SEO Tips for Web Developers</h1>
    </header>
    <main>
        <section>
            <h2>Understanding SEO Basics</h2>
            <p>
                Learn about the fundamental principles
                of SEO.
            </p>
            <a
                href="https://www.example.com/
                seo-basics"
            >
                Read more about SEO Basics
            </a>
        </section>
    </main>
</body>
```
- **Etiquetas canónicas**: Las etiquetas canónicas (`<link rel="canonical" href="URL"`) indican la versión preferida de una página. Ayudan a prevenir problemas de contenido duplicado, asegurando que los motores de búsqueda indexen la versión correcta. Un ejemplo:
```html
<head>
    <title>Top SEO Tips for Web Developers</title>
    <link
        rel="canonical"
        href=https://www.example.com/top-seo-tips
    >
</head>
```
El código HTML sirve como una representación fija de la dinámica de la aplicación para los motores de búsqueda y es uno de los elementos más impactantes en el SEO.
## CSS y SEO
Si bien no afecta directamente al SEO, tiene varios impactos indirectos:
- **Velocidad de carga de la página**: La forma en que se implementa CSS puede influir en los tiempos de carga de la página.
- **Compatibilidad móvil**: Los sitios web amigables con móviles se posicionan mejor en los resultados de búsqueda móvil, ya que los motores de búsqueda priorizan la usabilidad móvil.
- **Visibilidad del contenido**: Ocultar contenido con CSS (como `display:none`) puede ser visto con sospecha por los motores de búsqueda si parece manipulador, impactando potencialmente los rankings.
- **Fuente y legibilidad**: CSS controla la tipografía y el *layout*. Una mejor legibilidad y un *layout* limpio mejoran la interacción del usuario, beneficiando indirectamente al SEO a través de métricas de comportamiento del usuario mejoradas.
- **Uso de archivos CSS externos**: Los archivos CSS externos ayudan a mantener el HTML limpio y reducir el tamaño de la página. Un HTML limpio es más fácil de rastrear para los motores de búsqueda, y tamaños de página más pequeños mejoran los tiempos de carga.
- **Minificación y compresión**: Minificar y comprimir archivos CSS reduce su tamaño. Esta optimización mejora la velocidad de la página, lo cual es un factor de ranking conocido.

## Explorando las *Core Web Vitals*
Enfocarse en las *Core Web Vitals* es importante tanto para mejorar la experiencia del usuario como para lograr mejores rankings de búsqueda.
### *First Contentful Paint (FCP)*
Mide el tiempo que le toma al navegador renderizar la primera pieza de contenido (texto, imagen u otros elementos del DOM) en la pantalla desde la solicitud de navegación inicial.

Marca el punto en el proceso de carga cuando los usuarios pueden comenzar a ver contenido en la página web.

Los valores de 1.8 segundos o menos se consideran buenos.
![[Pasted image 20260131165750.png]]
Varios factores influyen en el FCP:
- **Tiempo de respuesta del servidor**: El tiempo que le toma al servidor responder a la solicitud inicial y comenzar a enviar recursos de vuelta.
- **Estructura HTML**: Un HTML estructurado eficientemente puede llevar a un análisis *(parsing)* y renderizado más rápido por parte del navegador.
- **CSS y JavaScript**: El CSS y JavaScript que bloquean el renderizado *(render-blocking)* pueden retrasar el FCP; optimizar y diferir el CSS y JavaScript no crítico puede mejorar el FCP.
- **Complejidad de la ruta de renderizado *(Render path)***: La cantidad de recursos (imágenes, scripts y hojas de estilo) y su tamaño impactan qué tan rápido el navegador puede renderizar el primer contenido significativo.
- **Condiciones de red**: La velocidad de red del usuario afecta qué tan rápido se pueden obtener y renderizar los recursos.
#### Técnicas para mejorar el FCP
Mejorar el FCP típicamente implica optimizar la entrega y el renderizado del contenido *above-the-fold* (la parte visible de la página sin hacer scroll). Aquí hay algunas estrategias:
- **Optimizar la ruta de renderizado crítica *(critical rendering path)***: Minimiza los recursos críticos reduciendo el número y tamaño de los archivos CSS y JavaScript:
	- Prioriza el contenido *above-the-fold* insertando en línea *(inlining)* el CSS esencial para un renderizado inmediato y difiriendo el CSS no esencial para que cargue asíncronamente, mejorando el rendimiento de la página.
	- Reduce los tiempos de respuesta del servidor usando una *Content Delivery Network* (CDN) y optimiza las configuraciones del servidor.
- **Optimizar HTML y CSS**: Elimina el CSS no utilizado quitando cualquier CSS que no se use para la vista inicial y minifica CSS y HTML.
- **Optimizar JavaScript**: Haces esto si usas minificación y carga asíncrona:
	- Minimiza el tamaño de los archivos JS y difiere el JS no crítico para que cargue después del contenido principal.
	- Usa carga asíncrona; carga los archivos JavaScript asíncronamente para prevenir que bloqueen el renderizado de la página.
- **Reducir recursos *render-blocking***: Carga los recursos críticos primero y asegura que los recursos esenciales se carguen rápidamente usando `defer` o `async` en el CSS y JS no crítico; esto asegura que el contenido inicial cargue más rápido.
### LCP *(Largest Contentful Paint)*
Mide el tiempo que tarda el elemento de contenido más grande en una página web en volverse visible dentro del *viewport* del usuario.

La métrica LCP considera elementos tales como los siguientes:
- Imágenes y elementos de imagen dentro de una etiqueta `<img>`.
- Imágenes de fondo cargadas vía CSS.
- Elementos de texto a nivel de bloque tales como `<p>` o `<h1>`.

Una buena puntuación de LCP es de 2.5 segundos o menos:

#### Técnicas para mejorar el LCP
- **Optimizar tiempos de respuesta del servidor**: Reduce los tiempos de respuesta del servidor usando un *hosting* web rápido, optimizando tu base de base de datos y usando una CDN. Implementa *caching* del lado del servidor para entregar páginas más rápidamente.
- **Usar prácticas de codificación eficientes**: Minifica CSS, JavaScript y HTML para reducir los tamaños de archivo. Elimina CSS y JavaScript innecesarios.
- **Optimizar imágenes y video**: Comprime imágenes sin sacrificar calidad usando herramientas como *ImageOptim* o *TinyPNG*. Sirve imágenes en formatos de próxima generación como *WebP*. Usa imágenes responsivas para asegurar que tengan el tamaño apropiado para diferentes dispositivos. Usa *lazy-load* para imágenes fuera de pantalla *(offscreen)* para diferir la carga hasta que sean necesarias.
- **Mejorar el renderizado del lado del cliente *(client-side rendering)***: Minimiza el tiempo de ejecución de JavaScript reduciendo el tamaño de tus *bundles* de JavaScript. Difiere el JavaScript no crítico y usa atributos `async` o `defer` para prevenir que bloquee el renderizado de la página.
- **Precargar recursos clave**: Usa la etiqueta `<link rel="preload">` para precargar recursos importantes como fuentes, CSS e imágenes para acelerar su carga.
- **Optimizar fuentes**: Usa `font-display: swap` en tu CSS para evitar texto invisible durante la carga de la fuente. Limita el uso de fuentes personalizadas y considera usar fuentes del sistema.
- **Implementar CSS crítico**: Extrae e inserta en línea  *(inline)* el CSS crítico (el CSS requerido para renderizar el contenido *above-the-fold*) directamente en el HTML para acelerar el renderizado.
- **Reducir scripts de terceros**: Limita el uso de scripts de terceros ya que pueden bloquear el hilo principal *(main thread)* y retrasar el LCP. Usa carga asíncrona para scripts de terceros cuando sea posible.
- **Reducir recursos de bloqueo**: Identifica y reduce recursos *render-blocking* como archivos grandes de CSS y JavaScript. Usa los atributos `async` o `defer` para archivos JavaScript que no son críticos para el renderizado inicial.
### FID *(First Input Delay)*
Mide el tiempo que le toma a una página web responder a la primera interacción del usuario. Esta interacción puede ser un clic, un toque o presionar una tecla.

**No mide el tiempo tomado para procesar el evento en sí, sino el tiempo que el navegador está ocupado haciendo otro trabajo y es incapaz des responder al usuario.**
#### Técnicas para mejorar el FID
- **Minimizar el trabajo del hilo principal (_main thread_):** Divide las tareas largas en tareas asíncronas más pequeñas para prevenir que el hilo principal sea bloqueado.
- **Optimizar la ejecución de JavaScript:** Minifica y comprime archivos JavaScript para reducir su tamaño. Elimina JavaScript no utilizado para evitar ejecución de código innecesaria. Divide grandes _bundles_ de JavaScript en fragmentos más pequeños y manejables usando técnicas de _code-splitting_.
- **Diferir JavaScript no crítico:** Usa los atributos `async` o `defer` en etiquetas `<script>` para cargar archivos JavaScript sin bloquear el hilo principal. Difiere la carga de scripts no esenciales hasta después de que la página haya sido renderizada inicialmente.
- **Reducir el impacto de código de terceros:** Limita el uso de scripts de terceros, ya que pueden introducir retrasos significativos. Asegúrate de que cualquier script de terceros se cargue asíncronamente.
- **Optimizar _event listeners_:** Evita usar _event listeners_ pasivos donde sea posible, ya que pueden retrasar la capacidad del navegador para responder a interacciones del usuario. Usa delegación de eventos para reducir el número de _event listeners_.
- **Usar prácticas de codificación eficientes:** Optimiza CSS para asegurar que no bloquee el renderizado. Inserta en línea (_inline_) el CSS crítico para acelerar el renderizado inicial de la página.
### CLS *(Cumulative Layout Shift)*
Mide la estabilidad visual de una página web, cuantifica cuánto se desplaza el contenido visible en el *viewport* mientras la página carga y se interactúa con ella.
#### Técnicas para mejorar el CLS
- **Siempre incluye atributos de tamaño en imágenes y videos:** Especifica atributos de `width` y `height` en las etiquetas `<img>` y `<video>` para asignar espacio en el _layout_ antes de que carguen los recursos.
- **Reserva espacio para anuncios e incrustados:** Asigna contenedores de tamaño fijo para anuncios y contenido incrustado. Asegúrate de que el espacio asignado no sea más pequeño que el contenido real.
- **Evita insertar contenido sobre contenido existente:** Evita insertar nuevo contenido por encima del contenido existente a menos que sea en respuesta a una acción del usuario. Por ejemplo, no añadas nuevos anuncios sobre el contenido principal de la página después de que la página haya cargado.
- **Usa CSS para asegurar _layouts_ estables:** Usa CSS para controlar animaciones y transiciones. Por ejemplo, prefiere animaciones de `transform` y `opacity` ya que no desencadenan cambios de _layout_. Asegúrate de que cualquier cambio de contenido dinámico no empuje el contenido existente fuera de lugar.
- **Precargar recursos importantes:** Usa `<link rel="preload">` para precargar recursos críticos, asegurando que se carguen rápidamente y reduciendo la probabilidad de cambios de _layout_.
- **Evitar el _jitter_ del _layout_:** Asegúrate de que los elementos interactivos como botones y formularios se rendericen en su estado final, minimizando cambios después de las interacciones del usuario.
## Cómo medir métricas de SEO
Varias herramientas y recursos pueden ayudarte a medir y analizar estas métricas esenciales:
- **Lighthouse**: Está integrado en las *Chrome DevTools* y proporciona información detallada sobre una gama de métricas de rendimiento (FCP, LCP, FID, CLS).
- **WebPageTest**: Ofrece un análisis en profundidad del rendimiento de carga, incluyendo desgloses detallados y visualizaciones para ayudarte a entender las secuencias de carga y los cuellos de botella, especialmente para FCP.
- **Web Vitals Extension**: Es una extensión de Chrome que mide las *Core Web Vitals* en tiempo real. Proporciona retroalimentación sobre qué tan rápido carga el contenido visible más grande (LCP), información sobre la capacidad de respuesta de la interacción (FID) y seguimiento en tiempo real de la estabilidad visual para ayudar a minimizar los cambios de _layout_ (CLS).
- **Google PageSpeed Insights**: Proporciona datos de rendimiento y sugerencias para mejorar. Se enfoca en analizar FCP y ofrece información accionable para mejorar el rendimiento de carga inicial.
- **Chrome User Experience Report**: Agrega datos de usuarios reales de usuarios de Chrome en todo el mundo. Proporciona datos del mundo real sobre qué tan rápido se vuelven visibles los grandes elementos de contenido (LCP) y datos de rendimiento de usuarios reales para ayudarte a entender y optimizar la latencia de interacción (FID). Para CLS, ofrece información sobre la estabilidad visual de tu sitio.
- **APIs de JavaScript:** Usando la interfaz `PerformanceObserver`, puedes rastrear FID programáticamente, permitiendo la recolección y análisis de datos personalizados. También rastrea CLS, permitiendo un seguimiento preciso de los cambios de _layout_.
## Minimizando recursos que bloquean el renderizado *(Render-blocking resources)*
Los recursos que bloquean el renderizado son archivos CSS y JavaScript que impiden que el navegador renderice la página hasta que son completamente cargados y ejecutados. Este retraso es particularmente notable para usuarios en conexiones más lentas o dispositivos menos potentes.

Veamos los tipos de recursos que bloquean el renderizado:
- **CSS:** Las hojas de estilo externas (enlazadas vía la etiqueta `<link>`) y las etiquetas `<style>` en línea (_inline_) pueden bloquear el renderizado porque el navegador necesita CSS para estilizar la página apropiadamente.
- **JavaScript:** Los scripts (enlazados vía la etiqueta `<script>`) que no están marcados como asíncronos (`async`) o diferidos (`defer`) pueden bloquear el renderizado porque el navegador debe ejecutarlos antes de continuar construyendo el DOM.
### Cómo minimizar recursos que bloquean el renderizado con CSS
Una técnica efectiva es optimizar la entrega de CSS. Esto implica dos estrategias clave: identificar e insertar en línea (_inline_) el CSS crítico y diferir el CSS no crítico.

El **CSS Crítico** se refiere al CSS mínimo requerido para renderizar el contenido _above-the-fold_ (la parte visible sin hacer _scroll_), que es la porción de la página web visible sin desplazarse. Al insertar en línea este CSS esencial, aseguras que el estilizado necesario se aplique rápidamente, mejorando el rendimiento de carga inicial.

Para el CSS no crítico, se recomienda cargar estos estilos asíncronamente después de que el contenido inicial de la página haya sido renderizado. Esto se puede lograr usando JavaScript para cargar hojas de estilo dinámicamente o empleando la etiqueta `<link rel="preload">` con el atributo `as="style"`.
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta
        name="viewport"
        content="width=device-width,
                initial-scale=1.0"
    >
    <title>Optimized Page</title>
    <style>
        /* Critical CSS */
        body {
            font-family: Arial, sans-serif;
            background-color: #f4f4f4;
        }
        header {
            background-color: #333;
            color: #fff;
            padding: 1em;
            text-align: center;
        }
    </style>
    <link
        rel="preload"
        href="styles/non-critical.css"
        as="style"
        onload="this.rel='stylesheet'"
    >
    <noscript>
        <link
            rel="stylesheet"
            href="styles/non-critical.css"
        >
    </noscript>
</head>
<body>
    <header>
        <h1>Welcome to My Website</h1>
    </header>
    <main>
        </main>
</body>
</html>
```
El elemento `<noscript>` se usa para proporcionar contenido de respaldo (_fallback_) para usuarios que tienen JavaScript deshabilitado en sus navegadores. En el contexto de diferir CSS no crítico, el bloque `<noscript>` asegura que el CSS no crítico se aplique incluso si JavaScript no está disponible.
### Cómo minimizar recursos que bloquean el renderizado con JavaScript
- Primero, el atributo `async` debería usarse para archivos JavaScript externos que no son esenciales para el renderizado inicial de la página. Esto permite que el script se descargue asíncronamente y se ejecute tan pronto como esté listo, sin bloquear otros recursos. Usa el atributo `async` para scripts que pueden correr independientemente de otros scripts y no dependen del DOM estando completamente cargado:
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta
        name="viewport"
        content="width=device-width,
                initial-scale=1.0"
    >
    <title>Optimized Page</title>
</head>
<body>
    <script
        src="path/to/non-essential-script.js"
        async
    ></script>
</body>
</html>
```
- Segundo, el atributo `defer` debería usarse para scripts que necesitan ejecutarse en orden pero no necesitan bloquear la construcción del DOM; los scripts con `defer` se ejecutan después de que el HTML ha sido completamente analizado (_parsed_). Podemos usar el atributo `defer` para scripts que necesitan ser ejecutados en orden y después de que el HTML ha sido analizado:
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta
        name="viewport"
        content="width=device-width,
                initial-scale=1.0"
    >
    <title>Optimized Page</title>
    <script src="path/to/script1.js" defer></script>
    <script src="path/to/script2.js" defer></script>
</head>
<body>
    </body>
</html>
```
- Adicionalmente, cargar JavaScript condicionalmente puede mejorar aún más el rendimiento. Esto incluye _lazy loading_ (carga diferida), lo cual retrasa la carga de JavaScript no crítico hasta que es necesario, como después de una acción del usuario o cuando se vuelve visible. Podemos hacer _lazy load_ de JavaScript no crítico para mejorar el rendimiento de carga inicial:
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta
        name="viewport"
        content="width=device-width,
                initial-scale=1.0"
    >
    <title>Optimized Page</title>
</head>
<body>
    <script>
        document.addEventListener(
            'DOMContentLoaded',
            function() {
                var lazyScript =
                    document.createElement('script');
                lazyScript.src =
                    'path/to/non-critical-script.js';
                document.body.appendChild(lazyScript);
            }
        );
    </script>
</body>
</html>
```
- La carga bajo demanda (_on-demand loading_) implica usar técnicas de carga de script dinámica, tales como `document.createElement('script')`, para cargar scripts solo cuando son necesarios. Deberíamos cargar scripts solo cuando son necesarios usando creación dinámica de scripts:
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta
        name="viewport"
        content="width=device-width,
                initial-scale=1.0"
    >
    <title>Optimized Page</title>
</head>
<body>
    <button id="loadScriptButton">Load Script</button>
    <script>
        document.getElementById(
            'loadScriptButton'
        ).addEventListener(
            'click',
            function() {
                var script =
                    document.createElement('script');
                script.src =
                    'path/to/conditional-script.js';
                document.body.appendChild(script);
            }
        );
    </script>
</body>
</html>
```
## Optimizando imágenes para SEO
- **Elige el formato de archivo correcto:**
	- **JPEG:** Mejor para fotografías e imágenes con muchos colores, JPEG ofrece un buen equilibrio entre calidad y tamaño de archivo.
	- **PNG:** Mejor para imágenes que requieren transparencia y para imágenes más simples con menos colores.
	- **WebP:** Un formato moderno que proporciona una compresión superior tanto para imágenes con pérdida como sin pérdida, soportado por la mayoría de los navegadores.
	- **SVG:** Mejor para logos, iconos y gráficos simples; es un formato vectorial, por lo que es escalable sin pérdida de calidad.
- **Comprime imágenes:** Usa herramientas como _TinyPNG_, _ImageOptim_ o _ShortPixel_ para reducir los tamaños de archivo de imagen sin comprometer la calidad. Asegúrate de que las imágenes estén apropiadamente comprimidas para disminuir los tiempos de carga, mejorando la velocidad de la página, lo cual es un factor de ranking.
- **Usa nombres de archivo descriptivos:** Renombra los archivos de imagen para incluir palabras clave relevantes antes de subirlos. En lugar de `IMG_1234.jpg`, usa `blue-widget-for-sale.jpg`. Usa guiones para separar palabras (no guiones bajos).    
- **Implementa imágenes responsivas:** Usa el atributo `srcset` para servir diferentes tamaños de imagen basados en el dispositivo del usuario, asegurando que las imágenes no sean más grandes de lo necesario. Implementa el elemento `<picture>` para ofrecer diferentes formatos de imagen.
- **Usa _lazy loading_:** Implementa carga diferida (_lazy loading_) para posponer la carga de imágenes fuera de pantalla hasta que el usuario se desplace cerca de ellas. Esto mejora el tiempo de carga inicial de la página. Usa el atributo `loading="lazy"` en etiquetas `<img>` o librerías de JavaScript para _lazy loading_.
- **Asegura que las imágenes sean rastreables:** Asegúrate de que las imágenes no estén bloqueadas por `robots.txt`. Usa mapas del sitio de imágenes (_image sitemaps_) para ayudar a los motores de búsqueda a descubrir e indexar imágenes en tu sitio.
- **Añade datos estructurados:** Usa datos estructurados (_schema.org_) para marcar imágenes con información adicional como detalles del producto, recetas y más. Esto puede mejorar los resultados de búsqueda con fragmentos enriquecidos (_rich snippets_) y mejorar la visibilidad.
- **Optimiza las dimensiones de la imagen:** Sirve imágenes en las dimensiones correctas para evitar el escalado del navegador. Esto asegura que las imágenes se vean nítidas y carguen más rápido. Dimensionar adecuadamente las imágenes para la web implica optimizar las imágenes para que se muestren en las dimensiones y tamaño de archivo correctos, lo que ayuda a mejorar la experiencia general del usuario.
```html
<img
    src="image.jpg"
    alt="Description of the image"
    width="800"
    height="600"
>
<img src="image.jpg"
    srcset="image-400.jpg 400w,
            image-800.jpg 800w,
            image-1200.jpg 1200w"
    sizes="(max-width: 600px) 400px,
           (max-width: 1000px) 800px,
           1200px"
    alt="Description of the image">
```
- **Aprovecha las CDNs:** Usa CDNs para distribuir contenido de imagen globalmente, reduciendo la latencia y mejorando los tiempos de carga para usuarios alrededor del mundo.
- **Usa pie de foto y contexto:** Proporciona pies de foto y contexto para las imágenes. Los motores de búsqueda usan el texto circundante para entender el contenido de la imagen. Imágenes bien descritas en contexto pueden posicionarse mejor en la búsqueda de imágenes.
- **Audita y actualiza regularmente:** Audita regularmente las imágenes de tu sitio para mejoras de rendimiento y SEO. Actualiza el texto alternativo (_alt text_), comprime imágenes y revisa enlaces rotos o contenido desactualizado.
### Usando etiquetas alt para imágenes
Necesitamos escribir texto alternativo (_alt text_) descriptivo que describa con precisión el contenido de la imagen e incluya palabras clave relevantes. El texto alternativo ayuda a los motores de búsqueda a entender el contexto de una imagen y mejora la accesibilidad para lectores de pantalla. 
#### Mejores prácticas para escribir etiquetas alt efectivas

Las etiquetas `alt` efectivas son cruciales para la accesibilidad y el SEO. Aquí hay algunas pautas para escribir mejores etiquetas `alt`:

- Deben ser descriptivas y específicas, describiendo con precisión el contenido y la función de la imagen sin detalles innecesarios.
- Usa palabras clave relevantes de forma natural, asegurando que el texto alternativo sea conciso y contextualmente relevante para el contenido circundante.
- Evita la redundancia y frases como "imagen de" o "foto de".
- Para imágenes funcionales, describe la función que sirven, mientras que las imágenes decorativas deberían tener un atributo `alt` vacío (`alt=""`).
- Revisa y actualiza regularmente el texto alternativo para mantener la precisión y utilidad.
## Evitando un tamaño de DOM excesivo
El tamaño del DOM se refiere al número total de elementos (etiquetas HTML), incluyendo nodos y sus relaciones (padre-hijo o hermano), estilos y atributos, que conforman la estructura de una página web.
### Técnicas para evitar un tamaño de DOM excesivo
- **Optimiza tu estructura HTML** simplificando el marcado. Usa HTML semántico y evita el anidamiento innecesario de elementos, manteniendo la estructura tan plana como sea posible. Elimina elementos, atributos y estilos en línea redundantes.
- **Limita el contenido dinámico** implementando _lazy loading_, que carga contenido dinámicamente solo cuando se necesita. También, usa técnicas como paginación o _infinite scroll_ para cargar contenido progresivamente a medida que el usuario se desplaza, en lugar de todo a la vez.
- **El uso eficiente de CSS y JavaScript** es también crucial. Minimiza la complejidad de CSS reduciendo el número de reglas y selectores CSS, y evita estilos excesivos que apliquen a muchos elementos. Optimiza JavaScript refactorizando código para minimizar operaciones de manipulación del DOM y mejorar el rendimiento.
- **Adicionalmente, optimiza el rendimiento de renderizado** priorizando la carga y renderizado de recursos críticos (CSS y JavaScript) necesarios para el contenido _above-the-fold_. Usa carga asíncrona (el atributo `async`) o ejecución diferida (el atributo `defer`) para scripts no críticos para mejorar la velocidad de renderizado inicial.
- **Monitorea y audita regularmente el tamaño del DOM** usando herramientas de desarrollador del navegador como _Chrome DevTools_ para inspeccionar y analizar el tamaño y estructura del DOM, identificando y abordando elementos o scripts que contribuyan excesivamente al tamaño del DOM y cuellos de botella de rendimiento.
- **Finalmente, considera el impacto de _frameworks_ y librerías de JavaScript** (como React, Angular o Vue.js) en el tamaño del DOM y el rendimiento, optimizando su uso y evaluando alternativas si es necesario.

Para más información sobre cómo el tamaño del DOM afecta la experiencia del usuario y estrategias para gestionarlo, revisa el artículo de Jeremy Wagner en web.dev (`https://web.dev/articles/dom-size-and-interactivity`). Adicionalmente, la documentación de Lighthouse ofrece una guía completa sobre reducción del tamaño del DOM: `https://developer.chrome.com/docs/lighthouse/performance/dom-size`.
