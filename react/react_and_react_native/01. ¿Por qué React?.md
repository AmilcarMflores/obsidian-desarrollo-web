## ¿Qué es React?
- React es una biblioteca de JavaScript para construir interfaces de usuario (ui).
- Se centra específicamente en la capa de vista (la parte que el usuario ve e interactúa).
- Se ocupa de renderizar y actualizar la UI en función de los cambios en los datos o el estado de la aplicación.
- Los componentes de React son como bloques de construcción que definen cómo se ve y se comporta cada parte de la UI.

Características principales:
### React component api
Los componentes son como funciones que devuelven elementos de la interfaz, pueden ser:
- Componentes de clase
- Componentes funcionales

Dentro de un componente manejamos:
- **Data**: información que el componente muestra o usa.
- **Lifecycle**: con hooks, usamos `useEffect` para manejar estos momentos.
- **Events**: cómo responde el componente a las acciones del usuario.
- **JSX**: sintaxis que parece HTML pero que en realidad es JavaScript.
### React DOM
Es el paquete que permite que React renderice componentes en el navegador, su método principal es `ReactDOM.render()`, que "pinta" el componente en el DOM real.

### Estructuras de UI declarativas con JSX
JSX (JavaScript XML) es una extensión de sintaxis que te permite escribir estructuras de UI dentro de JavaScript:
```jsx
function Saludo() {
	return <h1>¡Hola, React!</h1>
}
```
### Los datos cambian con el tiempo
Cada vez que los datos cambian, se vuelve a renderizar el componente, pero no se regenera toda la página, solo las partes que cambiaron.
### El rendimiento importa: el virtual DOM
El DOM real es la representación que el navegador hace de tu HTML, manipularlo directamente puede ser lento porque el navegador tiene que recalcular estilos, layout, etc.

React utiliza un virtual DOM, una representación en memoria del DOM real. 

Cuando algo cambia en tu componente:
1. React crea una nueva versión del virtual DOM.
2. Compara *(diffing)* la nueva versión con la anterior.
3. Identifica exactamente qué partes cambiaron.
4. Actualiza solo esas partes en el DOM real *(patching)*.

Esto hace que la actualizaciones sean muy eficientes.
## Configurando un nuevo proyecto de React
Existen varias formas de comenzar un proyecto en React, cada una con sus ventajas según el contexto del desarrollo.
### Node.js
- Antes de empezar, debes tener instalado Node.js (versión LTS recomendada).
- Node.js incluye `npm` (Node Package Manager), aunque usaremos `pnpm` (más rápido y eficiente en espacio).
### Uso de empaquetadores web
Ideal para aplicaciones de una sola página (SPA) y desarrollo local. Ofrece control total sobre la configuración:
#### Vite
Es una build-tool moderno, extremadamente rápido en desarrollo gracias a ES Modules nativos en el navegador.
##### Pasos para crear un proyecto con Vite + React
1. Crear el proyecto (desde la terminal, en la carpeta donde quieras guardarlo):
```bash
pnpm create vite my-react-app --template react
```
Esto crea una carpeta llamada `my-react-app` con la estructura básica de React.
2. Entrar al directorio del proyecto:
```bash
cd my-react-app
```
3. Instalar dependencias:
```bash
pnpm install
```
4. Iniciar el servidor de desarrollo:
```bash
pnpm run dev
```
Abre http://localhost:5173 (o el puerto que indique) en tu navegador.
### Usando framework (recomendado para proyectos más grandes o con SSR/SEO)
Estos frameworks vienen con funcionalidades listas para producción: enrutamiento, renderizado del lado del servidor (SSR), optimización de imágenes, etc.
#### Frameworks populares
| Framework   | Características clave                    | Ideal para                               |
| ----------- | ---------------------------------------- | ---------------------------------------- |
| **Next.js** | SSR, SSG, API Routes, File-based Routing | Aplicaciones full-stack, SEO, e-commerce |
| **Gatsby**  | SSG, GraphQL, Plugins                    | Sitios estáticos, blogs, marketing       |
| **Remix**   | SSR, Nested Routing, Web Standards       | Aplicaciones dinámicas con buen UX       |
Para crear un proyecto con Next.js:
```bash
pnpm create next-app my-next-app
```
### Editores de código en línea
Perfectos para experimentar, compartir prototipos o aprender sin instalar nada.
#### Plataformas populares
- **CodeSandbox**: Excelente integración con GitHub, plantillas listas.
- **StackBlitz**: Basado en VS Code, muy rápido, soporta Vite y Node.js en el navegador.
- **Replit**: Ideal para principiantes, permite colaboración en tiempo real.
