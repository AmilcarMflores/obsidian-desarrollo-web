## Pasar parámetros a un evento
El método `addEventListener` espera recibir una **referencia a una función** (la receta), no la ejecución de la función (el plato cocinado).
- El error común: `btn.addEventListener("click", saludar("Juan"))` esto ejecuta la función **inmediatamente** al cargar la página y devuelve `undefined` al evento.
- La solución (wrapper o envoltorio): Usamos una **función anónima (arrow function)** que envuelva nuestra llamada. El evento ejecuta la anónima, y la anónima ejecuta nuestra función con parámetros.
## Remover eventos (`removeEventListener`)
A veces necesitamos que un evento deje de escucharse (ej: un botón que solo funciona una vez o para liberar memoria).
- La regla de oro: Para remover un evento, la función manejadora **debe ser la misma referencia** en memoria.
- La trampa de las funciones anónimas: `btn.removeEventListener("click", () => { ... })` no funciona. Aunque el código dentro de la función sea idéntico, para JS son dos objetos funciones distintos.
- La solución: Debe guardar la función en una constante o variable (función nombrada) y pasar esa misma variable tanto al `add` como al `remove`.
