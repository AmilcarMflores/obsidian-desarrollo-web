Un evento es cualquier cosa que sucede en el documento (click, scroll, tecla presionada). Para reaccionar a ellos, necesitamos un **Manejador (Handler)**. Existen tres formas de definirlos:
## Atributo HTML (mala práctica)
Se escribe directamente en el HTML: `<button onclick="func()">`.
- Problema: Mezcla lógica (JS) con estructura (HTML). Ensucia el código y es difícil de mantener.
- Veredicto: Evítalo siempre.
## Manejador semántico (propiedad del DOM)
Se asigna una función a la propiedad del elemento en JS: `el.onclick = funcion`.
- Ventaja: Separa el JS del HTML.
- Gran desventaja: Solo soporta una función por evento. Si intentas asignar una segunda función, esta sobrescribirá a la primera.
- Veredicto: Útil para casos muy simples, pero limitado.
## Manejador múltiple (`addEventListener`)
Es el método estándar moderno.
- Ventaja: Permite ejecutar múltiples funciones para un mismo evento (puedes tener 3 funciones distintas reaccionado al mismo clic).
- Flexibilidad: Permite pasar opciones avanzadas (como ejecutar una vez, captura, etc.) y remover el evento con `removeEventListener`. 
- Veredicto: Úsalo siempre.
## El objeto evento (`e` o `event`)
Es un objeto especial que el navegador pasa automáticamente como único parámetro a la función manejadora. Contiene todos los detalles de lo que ocurrió:
- `type`: Tipo de evento (ej: "click")
- `target`: El elemento que originó el evento.
- `clientX`, `clientY`: Coordenadas del mouse, etc.
## Notas
- **El parámetro `e`**: Verás que a veces se llama `e`, `evt` o `event`. Es una convención llamarlo `e`. Acostúmbrate a declararlo siempre en tu función: `(e) => { ... }`.
    
- **`console.log(event)` vs `console.log(e)`**: En tu código original usabas `event` (la variable global de `window`). Aunque funciona en Chrome, es una práctica antigua (legacy de Internet Explorer). Lo profesional es usar el argumento `e` que recibe la función.
