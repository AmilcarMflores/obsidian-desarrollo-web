En el DOM, interactuar con los estilos tiene dos caras: lo que está escrito directamente en la etiqueta HTML ("inline") y lo que el navegador realmente muestra ("computed").
## `style` vs `getComputedStyle()`
1. `elemento.style`(Estilos en línea)
	- **Qué lee:** Solo accede a los estilos que están escritos en el atributo `style="..."` del HTML o que han sido añadidos mediante JS.
	- **Limitación:** No puede leer estilos definidos en hojas de CSS externas ni en etiquetas `<style>`. Si el estilo viene de un archivo `.css`, esta propiedad devolverá vacío.
	- **Uso:** Principalmente para escribir estilos.
2. `window.getComputedStyle(elemento)`
	- **Qué lee:** El valor final calculado por el navegador. Combina CSS externo, interno, inline y los estilos por defecto del navegador.
	- **Uso:** Solo para lectura (Read-only). Es lo que "se ve" realmente.
## Escritura de estilos
Existen dos sintaxis para aplicar estilos con JavaScript:
1. **Propiedad directa (CamelCase):** JS no permite guiones en nombres de variables/propiedades.
	- CSS: `background-color` ➡️ JS: `.style.backgroundColor`.
	- CSS: `margin-left` ➡️ JS: `.style.marginLeft`
2. **Método `setProperty` (texto exacto):** Permite usar el nombre real de CSS y es obligatorio para variables CSS (Custom Properties).
