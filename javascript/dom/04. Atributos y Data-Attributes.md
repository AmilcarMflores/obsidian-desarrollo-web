Podemos interactuar con los atributos de las etiquetas HTML (como `class`, `id`, `href`, `src`) de dos formas: usando **propiedades directas** (notación de punto) o usando los **métodos del DOM**.
## Atributos estándar (lectura y escritura)
- Notación de punto (`elemento.href`): Devuelve el valor **computado** por el navegador. En URLs, devuelve la ruta absoluta completa.
- Método (`getAttribute("href")`): Devuelve el valor **literal** (texto exacto) que está escrito en el HTML.
### Métodos principales
- `getAttribute("nombre")`: Obtiene el valor.
- `setAttribute("nombre", "valor")`: Crea o actualiza el valor.
- `hasAttribute("nombre")`: Devuelve `true` o `false` si existe.
- `removeAttribute("nombre")`: Elimina el atributo por completo del HTML.
## Data-Attributes (Atributos de datos)
- HTML5 introdujo los atributos `data-*` para guardar información extra en las etiquetas sin afectar el estilo ni la semántica. 
- JavaScript tiene una API especial para leerlos llamada `dataset`.
### Mapeo de nombres (CamelCase)
El navegador transforma automáticamente el formato:
- HTML: `data-description` ➡️ JS: `dataset.description`
- HTML: `data-id-usuario` ➡️ JS: `dataset.idUsuario`
## Notas
1. **Convención del signo de dólar (`$`)**: Es una convención muy común en la industria (heredada de jQuery) para indicar visualmente que esa variable guarda un **elemento del DOM** y diferenciarla de variables de datos normales (como un string o número).
2. **Seguridad en `target="_blank"`** Cuando uses `setAttribute("target", "_blank")`, siempre añade `rel="noopener noreferrer"`.  Esto evita una vulnerabilidad conocida como "Tabnabbing", donde la página nueva podría tomar control de la página original mediante JS malicioso.
3. **Cuándo usar `dataset` vs `getAttribute`**: Usa siempre `dataset` para leer/escribir `data-*`. Es más limpio y maneja automáticamente la conversión de nombres con guiones a camelCame.
