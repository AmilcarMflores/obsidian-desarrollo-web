Cuando queremos meter contenido dentro de una etiqueta, el resultado varía drásticamente según la propiedad que usemos. Supongamos que tenemos una cadena de texto que incluye etiquetas HTML (`<p>, <b>`).
## Inserción de texto plano
Estas propiedades escapan el HTML: las etiquetas se verán escritas en la pantalla en lugar de transformar el texto.
1. `textContent` (Estándar)
	- Comportamiento: Inserta el texto tal cual. Es rápido y eficiente.
	- Detalle: Recupera todo el texto, incluso el de elementos ocultos con CSS (`display: none`).
	- Uso: La opción por defecto recomendada.
2. `innerText` (No estándar / legacy)
	- Comportamiento: Fue creado por Internet Explorer. Aunque hoy funciona en todos los navegadores, es más lento.
	- Diferencia: Es "consciente del estilo"; si un elemento está oculto con CSS, `innerText` no lo leerá. Provoca un "reflow" (recalculo de diseño) en el navegador, lo que afecta el rendimiento.
## Inserción de HTML
3. `innerHTML`
	- Comportamiento: Analiza la cadena de texto y convierte las etiquetas en nodos del DOM reales dentro del elemento contenedor.
	- Seguridad: Cuidado con ataques XSS (Cross Site Scripting) si insertas datos que vienen de un usuario.
4. `outerHTML`
	- Comportamiento: Reemplaza al elemento mismo (incluyendo su etiqueta de apertura y cierre) por el nuevo contenido.
	- Efecto secundario: Después de ejecutar esto, la variable que referenciaba al elemento original (`$whatIsDOM`) sigue existiendo en memoria, pero el elemento ya no está en el documento (fue reemplazado).
